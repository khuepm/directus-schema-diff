# 🔍 Directus Schema Diff

> Công cụ so sánh cấu hình schema giữa các môi trường Directus — chạy hoàn toàn trên trình duyệt, không cần backend.

---

## 📋 Giới thiệu

Khi làm việc với Directus trên nhiều môi trường (Development, Staging, Production), các collection và field định nghĩa trong schema có thể bị lệch nhau theo thời gian.

Công cụ này giúp bạn:
- **Upload 2 file `schema.ts`** được export từ 2 môi trường khác nhau
- **Tự động phân tích** và so sánh sự khác biệt
- **Highlight rõ ràng** những gì đã thêm, xóa, hoặc thay đổi

---

## 🎯 Những gì được so sánh

Công cụ tập trung vào **2 phần quan trọng** trong file `schema.ts`:

### 1. `type Schema`
Danh sách tất cả các **collections** được định nghĩa trong dự án.

```typescript
export type Schema = {
  button: components["schemas"]["ItemsButton"][];
  media: components["schemas"]["ItemsMedia"][];
  // ...
};
```

### 2. `interface components { schemas }`
Định nghĩa chi tiết **cấu trúc field** của từng collection.

```typescript
export interface components {
  schemas: {
    ItemsButton: {
      id?: number;
      label?: string;
      url?: string | null;
    };
    // ...
  };
}
```

## 🚀 Cách sử dụng

### Bước 1: Mở công cụ
Mở file `index.html` trực tiếp trong trình duyệt (Chrome, Edge, Firefox).  
Không cần cài đặt, không cần server.

### Bước 2: Tạo file schema từ Directus

Directus cung cấp OpenAPI Specification tại endpoint `/server/specs/oas`. Dùng công cụ **openapi-typescript** để convert sang TypeScript.

#### 2.1. Cài đặt openapi-typescript

```bash
npm install -g openapi-typescript
# hoặc dùng npx (không cần cài đặt)
```

#### 2.2. Lấy Static Token từ Directus

File schema cần quyền **Admin** để đọc đầy đủ OpenAPI spec bao gồm custom collections.

1. Đăng nhập vào Directus Admin UI
2. Vào **Settings → Users** → mở user của bạn
3. Kéo xuống mục **Token** → tạo và copy token

> ⚠️ Nếu không truyền token, endpoint sẽ chỉ trả về schema của các system collection, thiếu custom collections.

#### 2.3. Generate file schema.ts

```bash
# Cú pháp cơ bản
npx openapi-typescript <DIRECTUS_URL>/server/specs/oas \
  --header "Authorization: Bearer <YOUR_TOKEN>" \
  -o schema.ts
```

**Ví dụ thực tế:**

```bash
# Production environment
npx openapi-typescript https://cms.example.com/server/specs/oas \
  --header "Authorization: Bearer your-prod-static-token" \
  -o schema-prod.ts

# Staging environment
npx openapi-typescript https://staging-cms.example.com/server/specs/oas \
  --header "Authorization: Bearer your-staging-static-token" \
  -o schema-staging.ts
```

**Trên Windows (Command Prompt):**

```cmd
npx openapi-typescript https://cms.example.com/server/specs/oas --header "Authorization: Bearer your-token" -o schema-prod.ts
```

**Trên Windows (PowerShell):**

```powershell
npx openapi-typescript https://cms.example.com/server/specs/oas `
  --header "Authorization: Bearer your-token" `
  -o schema-prod.ts
```

#### 2.4. Kiểm tra kết quả

File `schema-prod.ts` được tạo ra sẽ có dạng:

```typescript
// Generated by openapi-typescript
export interface paths { ... }
export interface components {
  schemas: {
    ItemsYourCollection: {
      id?: number;
      name?: string;
      // ...
    };
  };
}
export type Schema = {
  your_collection: components["schemas"]["ItemsYourCollection"][];
  // ...
};
```

> 💡 **Tip:** Nếu dùng Directus SDK, bạn có thể import trực tiếp `Schema` type này vào project TypeScript để có type-safety đầy đủ.

### Bước 3: Upload và so sánh
1. Kéo thả hoặc click **FILE NGUỒN (Source)** → chọn file môi trường cơ sở
2. Kéo thả hoặc click **FILE ĐÍCH (Target)** → chọn file môi trường cần kiểm tra
3. Nhấn **⚡ So sánh Schema**

---

## 🎨 Kết quả hiển thị

| Trạng thái | Ý nghĩa |
|------------|---------|
| ✅ **Added** | Collection / Field có trong Target nhưng không có trong Source |
| ❌ **Removed** | Collection / Field có trong Source nhưng bị mất ở Target |
| ✏️ **Modified** | Collection / Field tồn tại ở cả 2 nhưng có sự thay đổi về kiểu dữ liệu hoặc cấu trúc |
| 〰️ **Same** | Không có thay đổi |

### Tính năng
- 🔎 **Tìm kiếm** theo tên collection / schema
- 🗂️ **Filter** theo trạng thái (Added / Removed / Modified / Same)
- 📂 **Expand field-level diff** cho từng schema trong `components`
- 📊 **Summary cards** tổng hợp số lượng thay đổi
- 🖱️ **Drag & Drop** file trực tiếp vào panel

---

## 📝 Lưu ý

- File được xử lý **hoàn toàn trên máy client**, không có dữ liệu nào được gửi lên server
- Công cụ hỗ trợ file `.ts`, `.js`, `.txt`
- Với file schema lớn (> 1MB), quá trình phân tích có thể mất vài giây
